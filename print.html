German Covid Texts

Uploading New Documents

Each repository that you use should have its own subfolder in the raw folder, such as "google-scholar" or "pub-med".

Each document must be placed in the subfolder of the repository where it was found.

If such a folder does not exist yet, please create one.

Data Formats

The following formats are preferred, in descending order:

  1. XML or other strongly structured markup
  2. HTML or other semi-structured data
  3. Plain text
  4. PDF
  5. Images, scans and other visual representations

Data Processing

Text Extraction

Raw source data is processed using Apache Tika to extract plain text from any given file format.

Normalization

Tabs and duplicate whitespaces are removed from the plain text using regular expressions.

Segmentation & Tokenization

The plain text is segmented and tokenized using the Natural Language Toolkit.

Annotation Platform

We use version 22.1 of the annotation tool INCEpTION (Klie et al., 2018), hosted on internal servers at DFKI. The annotation platform is password-protected, but all annotations will be made publicly available in this repository.

Data Import

Inception is able to import plain text data from .txt files where each segment is on a new line and tokens are separated by a single whitespace.

Data Export

Data Format

Annotations are exported to XML files based on the UIMA CAS XMI specification.

Standoff Annotations

Additional layers and their annotations are serialized in a standoff manner, i.e., with references to their respective base layer (such as IDs of tokens or segments).

References

Klie, J.-C., Bugert, M., Boullosa, B., de Castilho, R. E., & Gurevych, I. (2018, Juni). The INCEpTION Platform: Machine-Assisted and Knowledge- Oriented Interactive Annotation. In Proceedings of the 27th International Conference on Computational Linguistics: System Demonstrations (pp. 5–9). Santa Fe, New Mexico: Association for Computational Linguistics. Annotation Schemas

All schemas sorted by layer:

  • Raw Text
  • Segment
  • Token
  • Credibility
  • Political Bias
  • Named Entities
  • Relations
  • Questions
  • Answers
  • Contexts

Raw Text

The raw text forms the basic layer for the whole dataset. It should be in a plain text format: a single string, without any visual formatting.

Other layers that directly reference this layer:

  • Segment
  • Token

Segment

The segment layer points to character indices in the raw text layer. It marks spans of characters as belonging to the same segment using unique identifiers.

Other layers that directly reference this layer:

  • Credibility
  • Political Bias
  • Questions

Token

The token layer points character indices in the raw text layer. It marks spans of characters as belonging to the same token using unique identifiers.

Other layers that directly reference this layer:

  • Named Entities
  • Answers

Credibility

The credibility layer points to segment identifiers. It marks single segments as having either high or low credibility.

Political Bias

The political bias layer points to token identifiers. It marks single tokens as belonging to one of the 5 classes described in this paper:

Aksenov, D., Bourgonje, P., Zaczynska, K., Ostendorff, M., Moreno-Schneider, J., & Rehm, G. (2021). Fine-grained Classification of Political Bias in German News: A Data Set and Initial Experiments. Proceedings of the 5th Workshop on Online Abuse and Harms (WOAH 2021), 121–131. https://doi.org/10.18653/v1/2021.woah-1.13

In that paper, the classes are:

  • far-left
  • center-left
  • center
  • center-right
  • far-right

The classes can be collapsed into a binary setup for detection of hyperpartisanship: far-left and far-right are hyperpartisan, the others are non-hyperpartisan.

Named Entities

The named entity layer points to token identifiers. It marks spans of tokens as belonging to one of the 4 classes described in OpeNER:

  • Person
  • Location
  • Organization
  • Miscellaneous

Other layers that directly reference this layer:

  • Relations

Unnamed Entities

Entities without a name are not annotated in this layer.

Example

"generell kann man nie sagen wer schon vielleicht Antikörper hat"

Coreference Resolution

Coreferences through pronouns or other pointers ("") are not resolved in this layer. Therefore, they do not get annotated as entities, due to our policy on Unnamed Entities.

Example

50% meiner Bekannten die sich Impfen lassen haben und vorher kein Leiden hatten sind jetzt zum Teil Arbeitsunfähig und Krank

Relations

The relation layer points to entity identifiers. It marks two Named Entities as connected by a WikiData property.

Questions

The question layer points to segment identifiers. It marks a segment as containing a question.

Answers

The answer layer points to token identifiers. It marks a span of tokens as containing an answer to one or more questions.

Contexts

The context layer points to segment identifiers. It marks a segment as containing contextually relevant content for a question-answer pair.
